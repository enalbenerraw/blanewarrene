# Roadmap Map Template

A single combined view of both product roadmaps. Use this before any conflict diagnosis. The map exists to create shared visibility, not to drive decisions.

## Structure

| Theme | Acquirer epic | Acquired epic | Customer outcome | Dependency | Status both sides |
|-------|---------------|---------------|------------------|------------|--------------------|
| | | | | | |

## Themes to consider

Use whatever themes the two teams already use, but ensure these are represented:

- Core product capability
- Platform and infrastructure
- API and ecosystem
- Compliance and security
- Pricing and packaging adjacencies
- Customer-requested features (named accounts)

## Dependency notation

- **Hard dependency**: Item B cannot ship until Item A ships
- **Soft dependency**: Item B is meaningfully better if Item A ships first
- **Conflict**: Items A and B do the same thing differently and one must give

## Visual layout (if used in a whiteboard tool)

- X-axis: time (next 30 / 60 / 90 / 12 months)
- Y-axis: themes
- Cards: epics, color-coded by team (acquirer vs acquired)
- Lines: dependencies and conflicts
