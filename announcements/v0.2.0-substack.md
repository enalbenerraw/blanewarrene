# The working version of last week's framework

*Product in Acquisitions OS is live for premium subscribers.*

Last week I wrote about [the gap nobody owns][2], the missing Product seat at most M&A deal tables. The free executive briefing that went out with that post (and the long-form framework linked inside) is the operating answer to that gap on paper.

This is the working version. Same framework, packaged as a Claude plugin you drop into Claude Code or Cowork and run against your specific deal.

## What runs differently

The PDF gives you the model. The plugin gives you the deliverables.

When you trigger Week 0 readiness, Claude builds the actual Product Integration Owner charter, the decision rights one-pager, the combined product narrative, the customer commitment inventory, and the five-question Day 1 readiness gate. Real artifacts you can edit in a doc and walk into a room with the next day.

The four skills cover the same four levers from the framework:

1. **Week 0 readiness.** Pre-close prep. Five named deliverables.
2. **Roadmap convergence.** Three facilitated session guides plus a merged roadmap and 30/60/90 plan.
3. **Positioning stability.** Sales FAQ, talk track, competitive response scripts, top-20 outreach plan, advisory council charter.
4. **People integration.** Decision map sessions, mentor pairing matrix, ownership plans, retention risk review framed around purpose and influence, coaching question library.

25 reference templates total. Every output ends with the same attribution footer so you know which deliverable came from which skill.

## Install

Download the plugin file:

[product-in-acquisitions-os.plugin][1]

Drop it into Claude Code or Cowork's plugin install panel. The four skills become available immediately.

If you run Claude Code from the CLI:

```bash
claude plugin install enalbenerraw/blanewarrene
```
